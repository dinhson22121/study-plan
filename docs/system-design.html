1.Giới thiệu & mục tiêu

AI Study Coach giúp học sinh THPT (15–18 tuổi) có lộ trình học rõ ràng: chẩn đoán đầu vào → sinh study plan → học theo chủ đề → làm quiz ngắn → theo dõi tiến độ, kèm nhắc học qua push notification. Admin upload tài liệu PDF, hệ thống tự parse thành câu hỏi nháp để review & publish.

Nguyên tắc thiết kế

  • Modular monolith theo DDD — 12 domain module tách biệt, giao tiếp qua ports & adapters và domain events; deploy 1 binary, dễ tách microservice sau.
  • Event-driven cho việc phụ (progress, analytics, notification) — không chặn luồng chính của người dùng.
  • Stateless API — scale ngang dễ; state nằm ở Postgres/Redis/Kafka/S3.
  • Security & observability từ đầu — JWT rotation+revocation, rate limit, audit admin, metrics, Sentry, structured log.

5.Phân tầng DDD (mỗi module)

Giao tiếp giữa module dùng ports & adapters: ví dụ studyplan đọc curriculum/placement/goal qua adapter; analytics phản ứng sự kiện quiz.completed. Container app.Deps chứa các handoff: AuthValidate, Notifier, ReengagementSource, Metrics.

6.Bản đồ 12 domain module

auth

Đăng ký/đăng nhập, JWT access+refresh (rotation, revocation), bcrypt.

user

Hồ sơ người dùng; tạo khi user.registered.

curriculum

Subject → Chapter → Topic (có cache).

content

Lesson theo topic; admin upload + parse (có cache).

question

Ngân hàng câu hỏi; draft từ parse → publish.

goal

Mục tiêu học & thời lượng/ngày.

placement

Bài chẩn đoán đầu vào, kết quả theo môn.

studyplan

Sinh lộ trình + milestone từ goal/placement.

quiz

Bắt đầu/nộp/chấm quiz, ownership-scoped.

progress

Streak, % hoàn thành; cập nhật theo event.

analytics

Weak topics, activity, inactive users.

notification

FCM push, preference, broadcast, re-engagement.

7.Domain events & Kafka

Việc phụ chạy bất đồng bộ qua in-process EventBus (đồng bộ, best-effort, nuốt lỗi để không làm hỏng hành động người dùng) và Kafka cho notification pipeline.

TopicÝ nghĩa
notification.scheduleLịch nhắc học / re-engagement được xếp hàng
notification.sendYêu cầu gửi push tới FCM (qua preference gate + idempotency)
notification.resultKết quả gửi (thành công/thất bại, token invalid)
notification.dlqDead-letter: message không decode/không xử lý được
Ví dụ event nội bộ: user.registered → tạo profile; quiz.completed → cập nhật progress + analytics (activity ping). Idempotency dùng Redis SETNX; consumer dùng WaitGroup để shutdown sạch.

14.Cross-cutting concerns

Bảo mật

bcrypt; JWT HS256 pinned; refresh rotation+revocation; access-token blocklist (jti); password policy 10–72; prod config validation (sslmode/secret).

Rate limiting

Redis fixed-window (Lua atomic): auth 10/phút, toàn /api/v1 120/phút per-IP; trusted-proxy cho X-Forwarded-For; 429.

Cache

Cache-aside (Redis) cho curriculum + lessons; invalidate khi admin ghi; TTL 10 phút.

Observability

Prometheus /metrics (latency, 4xx/5xx, failures); Sentry (gated DSN); structured log + correlation-id.

Audit

admin_audit_log ghi mọi mutation của ADMIN (actor/method/path/status).

Reliability

Graceful shutdown; readiness probe (DB+Redis); transaction cho quiz/placement submit; Kafka DLQ.

15.Deployment

16.Non-functional & scaling

  • Scale ngang: API stateless → tăng instance sau ALB; Redis/Kafka/RDS managed.
  • Hiệu năng: cache-aside giảm tải DB cho dữ liệu tham chiếu; index theo truy vấn nóng; pgxpool.
  • Tính sẵn sàng: readiness/liveness, graceful shutdown, restart policy, RDS Multi-AZ (toggle).
  • Bảo mật: TLS, secret qua SSM, S3 private, rate limit, audit; 0 vuln (govulncheck/npm/pip-audit).
  • Khả năng quan sát: metrics + Sentry + correlation-id xuyên suốt.

17.Giới hạn hiện tại & roadmap

  • Push thật cần cấu hình Firebase (hiện fallback log nếu thiếu).
  • Audit chi tiết cho hành động mobile/student chưa có (chỉ activity ping khi quiz xong) — Phase sau.
  • Diagnostic flow mobile: start test → đọc kết quả (submit câu hỏi là mở rộng).
  • Kafka TLS/SASL, OCR cho PDF scan, iOS release, advanced analytics — ngoài scope Phase 1.
  • Đi public cần: apply Terraform thật, closed beta (crash-free ≥ 99.5%), đóng các mục infra trong docs/SECURITY_CLOSURE.md.
